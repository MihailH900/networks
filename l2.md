# Коммутация L2 и ограничения портов

Основная реализация выполняется на MikroTik через WebFig. Для Rapier приведены отдельные команды AlliedWare.

Перед началом выполните подготовку из [`00-basic-setup`](../00-basic-setup/).

Обозначения:

* `<BRIDGE>` — имя bridge;
* `<CLIENT_PORT>` — клиентский порт;
* `<SECOND_PORT>` — второй клиентский порт;
* `<MIRROR_PORT>` — порт компьютера, принимающего копии трафика;
* `<MAC>` — MAC-адрес устройства;
* `<IP>` — IPv4-адрес;
* `<INTERFACE>` — L3-интерфейс, bridge или VLAN-интерфейс.

В примерах используются:

```text
bridge:         br-l2
клиент 1:      ether2
клиент 2:      ether3
порт зеркала:  ether4
управление:    ether10
```

## 1. Обычный L2-коммутатор

### 1.1. Схема

```text
PC-A ─── ether2 MikroTik ether3 ─── PC-B

ether10 ─── компьютер администратора
```

IP-адреса компьютеров:

| Устройство | Адрес              |
| ---------- | ------------------ |
| PC-A       | `192.168.10.10/24` |
| PC-B       | `192.168.10.20/24` |

Шлюз не нужен: оба компьютера находятся в одной сети.

### 1.2. MikroTik через WebFig

#### Шаг 1. Проверить порты

Откройте:

> **Bridge → Ports**

Убедитесь, что `ether2` и `ether3`:

* не входят в другой bridge;
* не используются в bonding;
* не имеют собственных IP-адресов.

#### Шаг 2. Создать bridge

Откройте:

> **Bridge → Bridge → Add New**

Укажите:

| Поле           | Значение |
| -------------- | -------- |
| Name           | `br-l2`  |
| Protocol Mode  | `rstp`   |
| VLAN Filtering | `no`     |

Нажмите **Apply**, затем **OK**.

#### Шаг 3. Добавить первый порт

Откройте:

> **Bridge → Ports → Add New**

Укажите:

| Поле             | Значение |
| ---------------- | -------- |
| Interface        | `ether2` |
| Bridge           | `br-l2`  |
| Edge             | `yes`    |
| Hardware Offload | `yes`    |

#### Шаг 4. Добавить второй порт

Повторите настройку для `ether3`.

#### Шаг 5. Настроить компьютеры

Назначьте:

```text
PC-A: 192.168.10.10/24
PC-B: 192.168.10.20/24
```

#### Проверка

С PC-A:

```bash
ping 192.168.10.20
```

MikroTik выполняет только коммутацию. IP-адрес на `ether2`, `ether3` или `br-l2` для этого не требуется.

### 1.3. Rapier

В исходной конфигурации обычные порты уже находятся в одной default VLAN.

Через веб-интерфейс откройте:

> **Configuration → Port → Settings**

Для нужных портов установите:

```text
Enabled: yes
Speed: Auto
```

Через CLI:

```text
enable switch port=2,3
set switch port=2,3 speed=autonegotiate
```

Подключите компьютеры к портам 2 и 3 и назначьте им адреса одной подсети.

## 2. Скорость, дуплекс и автосогласование

По умолчанию используйте автосогласование. Ручной режим нужен только по условию задания.

### 2.1. MikroTik: автоматический режим

Откройте:

> **Interfaces → Ethernet → `<PORT>`**

Установите:

```text
Auto Negotiation: yes
```

Если присутствует поле **Advertise**, оставьте все поддерживаемые режимы либо только разрешённые условием.

Текущие параметры отображаются в состоянии интерфейса:

```text
Status: link-ok
Rate: 1 Gbps
Full Duplex: yes
```

### 2.2. MikroTik: ручной режим

Для проверки режима `100 Мбит/с full-duplex`:

> **Interfaces → Ethernet → `<PORT>`**

Установите:

```text
Auto Negotiation: no
Speed: 100M-baseT-full
```

На старой версии RouterOS значения могут называться:

```text
100Mbps
Full Duplex: yes
```

Тот же режим нужно настроить на втором конце линии.

Не отключайте автосогласование только с одной стороны.

Для обычного медного Gigabit Ethernet оставляйте автосогласование включённым.

### 2.3. Настройка Windows

Откройте:

> **Сетевые подключения → Ethernet → Свойства → Настроить → Дополнительно**

Найдите:

```text
Speed & Duplex
```

Выберите:

```text
Auto Negotiation
```

или заданный режим:

```text
100 Mbps Full Duplex
```

### 2.4. Настройка Linux

Графический интерфейс обычно не позволяет менять дуплекс. Используйте `ethtool`.

Ручной режим:

```bash
sudo ethtool -s <INTERFACE> speed 100 duplex full autoneg off
```

Возврат к автоматическому режиму:

```bash
sudo ethtool -s <INTERFACE> autoneg on
```

### 2.5. Rapier

Через веб-интерфейс:

> **Configuration → Port → Settings**

Выберите нужную скорость и дуплекс.

Через CLI:

```text
set switch port=2 speed=autonegotiate
```

Ручные режимы:

```text
set switch port=2 speed=100Mfull
set switch port=2 speed=100Mhalf
set switch port=2 speed=10Mfull
set switch port=2 speed=10Mhalf
```

После эксперимента верните:

```text
set switch port=2 speed=autonegotiate
```

## 3. FDB и обучение MAC-адресам

FDB хранит соответствия:

```text
MAC-адрес → порт коммутатора
```

### 3.1. Динамическое обучение на MikroTik

Откройте:

> **Bridge → Hosts**

После обмена между PC-A и PC-B должны появиться записи:

| MAC      | On Interface | Bridge  |
| -------- | ------------ | ------- |
| MAC PC-A | `ether2`     | `br-l2` |
| MAC PC-B | `ether3`     | `br-l2` |

Если записи нет, отправьте любой трафик с компьютера, например `ping`.

### 3.2. Статическая запись MAC → порт на MikroTik

Откройте:

> **Bridge → Hosts → Add New**

Укажите:

| Поле        | Значение                 |
| ----------- | ------------------------ |
| MAC Address | `<MAC>`                  |
| Bridge      | `br-l2`                  |
| Interface   | `ether2`                 |
| VID         | оставить пустым без VLAN |

После этого кадры к указанному MAC будут направляться через `ether2`.

Если используется VLAN Filtering, дополнительно задайте правильный `VID`.

### 3.3. Отключение обучения на порту MikroTik

Откройте:

> **Bridge → Ports → `ether2`**

Установите:

```text
Learn: no
```

При необходимости также:

```text
Unknown Unicast Flood: no
```

Перед отключением обучения добавьте статические записи для разрешённых MAC-адресов. Иначе коммутатор не будет знать, куда отправлять обычный unicast-трафик.

### 3.4. Очистка динамических записей MikroTik

Откройте:

> **Bridge → Hosts**

Выберите динамические записи и нажмите **Remove** или **Flush**, если такая кнопка доступна в используемой версии WebFig.

Статические записи удаляются вручную.

### 3.5. Rapier

Просмотр FDB:

```text
show switch FDB
```

Добавление статической записи:

```text
add switch filter action=forward destaddress=<MAC> port=<PORT>
```

Пример:

```text
add switch filter action=forward destaddress=00-11-22-33-44-55 port=2
```

Запрещающая запись по MAC назначения:

```text
add switch filter action=discard destaddress=<MAC> port=<PORT>
```

Остановка дальнейшего обучения на порту:

```text
activate switch port=<PORT> lock
```

Пример:

```text
activate switch port=2 lock
```

Для удаления фильтра используйте:

```text
delete switch filter destaddress=<MAC>
```

Если версия AlliedWare требует дополнительные параметры, выполните:

```text
delete switch filter ?
```

<a id="static-arp"></a>

## 4. Статический ARP

Статический ARP связывает:

```text
IPv4-адрес → MAC-адрес
```

Это L3-настройка. Она не заменяет Port Security и не запрещает использование порта другим MAC-адресом.

### 4.1. MikroTik

Откройте:

> **IP → ARP → Add New**

Укажите:

| Поле        | Значение      |
| ----------- | ------------- |
| IP Address  | `<IP>`        |
| MAC Address | `<MAC>`       |
| Interface   | `<INTERFACE>` |

Интерфейсом должен быть тот L3-интерфейс, через который доступен узел:

* bridge;
* VLAN-интерфейс;
* отдельный физический L3-порт.

Пример:

```text
IP Address: 192.168.10.20
MAC Address: 00:11:22:33:44:55
Interface: br-l2
```

### 4.2. Разрешение только статических ARP-записей

Откройте интерфейс, на котором находится IP-подсеть:

> **Interfaces → `<INTERFACE>`**

Установите:

```text
ARP: reply-only
```

После этого заранее добавьте статические ARP-записи для всех разрешённых соседей.

Не включайте `reply-only`, если таблица ARP ещё не заполнена.

### 4.3. Rapier

Статический ARP нужен только в том случае, если Rapier выполняет функции L3-коммутатора.

Пример:

```text
add ip arp=<IP> interface=<VLAN_INTERFACE> ethernet=<MAC> port=<PORT>
```

Например:

```text
add ip arp=192.168.10.20 interface=vlan10 ethernet=00-11-22-33-44-55 port=2
```

Просмотр:

```text
show ip arp
```

Удаление:

```text
delete ip arp=<IP>
```

<a id="port-security"></a>

## 5. Port Security

Port Security применяется к клиентским портам:

```text
компьютер
касса
киоск
принтер
обычный сервер
```

Не применяйте ограничение одним MAC к:

```text
trunk
LACP
точке доступа
виртуализационному серверу
порту другого коммутатора
```

### 5.1. MikroTik: разрешение одного известного MAC

На RB1100AHx4 строгая фильтрация выполняется программным bridge filter.

Схема:

```text
разрешённый MAC → пропустить
любой другой MAC с этого порта → отбросить
```

#### Шаг 1. Узнать разрешённый MAC

Откройте:

> **Bridge → Hosts**

Скопируйте MAC устройства на `ether2`.

#### Шаг 2. Отключить аппаратный обход фильтра

Откройте:

> **Bridge → Ports → `ether2`**

Установите:

```text
Hardware Offload: no
Learn: no
```

#### Шаг 3. Добавить статическую FDB-запись

Откройте:

> **Bridge → Hosts → Add New**

Укажите:

```text
MAC Address: <MAC>
Bridge: br-l2
Interface: ether2
```

#### Шаг 4. Разрешить MAC для транзитного трафика

Откройте:

> **Bridge → Filter → Add New**

Первая запись:

| Поле             | Значение  |
| ---------------- | --------- |
| Chain            | `forward` |
| In. Interface    | `ether2`  |
| Src. MAC Address | `<MAC>`   |
| Action           | `accept`  |

#### Шаг 5. Заблокировать остальные MAC

Добавьте вторую запись:

| Поле          | Значение  |
| ------------- | --------- |
| Chain         | `forward` |
| In. Interface | `ether2`  |
| Action        | `drop`    |

Разрешающее правило должно находиться выше запрещающего.

#### Шаг 6. Защитить доступ к самому MikroTik

Если bridge имеет IP-адрес, DHCP-сервер или другие службы, добавьте две аналогичные записи для цепочки `input`.

Разрешение:

```text
Chain: input
In. Interface: ether2
Src. MAC Address: <MAC>
Action: accept
```

Запрет:

```text
Chain: input
In. Interface: ether2
Action: drop
```

### 5.2. MikroTik: разрешение нескольких MAC

Для каждого разрешённого MAC добавьте отдельное правило `accept`.

Пример порядка:

```text
1. accept MAC-A
2. accept MAC-B
3. accept MAC-C
4. drop всё остальное с ether2
```

Так реализуется ограничение до заранее известных устройств.

### 5.3. MikroTik: разрешить первый изученный MAC

На RB1100AHx4 это выполняется вручную:

1. оставьте `Learn: yes`;
2. подключите разрешённый компьютер;
3. создайте трафик;
4. скопируйте его MAC из **Bridge → Hosts**;
5. создайте статическую запись;
6. установите `Learn: no`;
7. создайте разрешающее и запрещающее правила.

Это ручной аналог `lock on first`.

### 5.4. Реакция MikroTik на нарушение

На RB1100AHx4 основной вариант:

```text
Action: drop
```

При необходимости можно включить логирование отдельным правилом перед `drop`.

Автоматический перевод неизвестного устройства в отдельную VLAN выполняется не обычным Port Security, а через RADIUS/MAC Authentication. См. методичку:

```text
05-radius-access-control
```

### 5.5. Rapier: ограничение числа MAC

Разрешить один изученный MAC:

```text
set switch port=2 learn=1 intrusionaction=discard
```

Разрешить до трёх MAC:

```text
set switch port=2 learn=3 intrusionaction=discard
```

Действия при нарушении:

| Значение  | Результат                      |
| --------- | ------------------------------ |
| `discard` | кадры нарушителя отбрасываются |
| `trap`    | отправляется SNMP Trap         |
| `disable` | порт отключается               |

Пример с отключением порта:

```text
set switch port=2 learn=1 intrusionaction=disable
```

Зафиксировать уже изученные MAC и остановить обучение:

```text
activate switch port=2 lock
```

Просмотр нарушителей:

```text
show switch port=2 intrusion
```

Просмотр FDB:

```text
show switch FDB
```

### 5.6. Rapier: разрешение конкретного MAC

Добавьте статическое разрешающее правило:

```text
add switch filter action=forward destaddress=<MAC> port=2
```

Затем запретите обучение новых адресов:

```text
set switch port=2 learn=none
activate switch port=2 lock
```

Для полноценной проверки source MAC предпочтительнее использовать встроенный `learn` и `intrusionaction`, а не только фильтр по адресу назначения.

## 6. Port Mirroring

Port Mirroring создаёт копию трафика исходного порта и передаёт её на отдельный monitor-порт.

Схема:

```text
PC-A ─── ether2 MikroTik

PC-Monitor ─── ether4 MikroTik
```

`ether2` — source.

`ether4` — target.

### 6.1. Подготовка monitor-порта MikroTik

Откройте:

> **Bridge → Ports**

Удалите `ether4` из обычного рабочего bridge.

Monitor-порт не должен:

* использоваться как клиентский порт;
* входить в LACP;
* создавать обратный трафик в исследуемую сеть.

### 6.2. Определение switch chip

Откройте:

> **Switch → Ports**

Определите, к какому switch chip относятся:

```text
ether2
ether4
```

Source и target должны принадлежать одному switch chip.

### 6.3. MikroTik: зеркалирование физического порта

Откройте:

> **Switch → Switch**

Выберите нужный switch chip и укажите:

```text
Mirror Source: ether2
Mirror Target: ether4
```

После применения весь входящий и исходящий трафик `ether2` копируется на `ether4`.

На RB1100AHx4 с RTL8367 отдельно выбрать только RX или только TX нельзя: аппаратное зеркалирование копирует оба направления.

### 6.4. Зеркалирование trunk или серверного порта

В поле:

```text
Mirror Source
```

укажите физический порт trunk или физический серверный порт.

Если trunk реализован через два порта LACP, обычное аппаратное зеркалирование одного slave-порта покажет только трафик, проходящий именно через него. Для полного наблюдения агрегата может потребоваться поочерёдно зеркалировать каждый slave.

### 6.5. Отключение зеркалирования MikroTik

Откройте:

> **Switch → Switch**

Установите:

```text
Mirror Source: none
Mirror Target: none
```

После этого monitor-порт можно снова использовать в обычной конфигурации.

### 6.6. Rapier: зеркалирование обоих направлений

Включите функцию:

```text
enable switch mirror
```

Назначьте monitor-порт:

```text
set switch mirror=4
```

Назначьте source-порт:

```text
set switch port=2 mirror=both
```

### 6.7. Rapier: только входящий трафик

```text
set switch port=2 mirror=rx
```

### 6.8. Rapier: только исходящий трафик

```text
set switch port=2 mirror=tx
```

### 6.9. Несколько source-портов Rapier

```text
set switch port=2,3,5 mirror=both
```

Все копии будут направляться на один monitor-порт.

### 6.10. Отключение зеркалирования Rapier

```text
set switch port=2,3,5 mirror=none
set switch mirror=none
disable switch mirror
```

Monitor-порт Rapier используется только для передачи копий трафика и не должен применяться как обычный клиентский порт во время зеркалирования.

<a id="cleanup"></a>

## 7. Очистка конфигурации

### 7.1. MikroTik

Удаляйте настройки в таком порядке:

1. отключите Port Mirroring;
2. удалите правила **Bridge → Filter**;
3. удалите статические записи **Bridge → Hosts**;
4. верните `Learn: auto` или `yes`;
5. верните `Hardware Offload: yes`;
6. удалите статические записи **IP → ARP**;
7. верните `ARP: enabled`, если использовался `reply-only`;
8. удалите учебные порты из bridge;
9. удалите `br-l2`, если он больше не нужен;
10. верните Ethernet-портам `Auto Negotiation: yes`.

Не удаляйте management bridge и management-порт.

### 7.2. Rapier

Возврат скорости:

```text
set switch port=2,3 speed=autonegotiate
```

Отключение зеркалирования:

```text
set switch port=2,3 mirror=none
set switch mirror=none
disable switch mirror
```

Снятие ограничений обучения:

```text
set switch port=2,3 learn=256 intrusionaction=discard
```

Удаление статических фильтров:

```text
delete switch filter destaddress=<MAC>
```

Удаление статического ARP:

```text
delete ip arp=<IP>
```

<a id="mapping"></a>

## 8. Соответствие MikroTik и Rapier

| Задача                         | MikroTik WebFig                   | Rapier                             |
| ------------------------------ | --------------------------------- | ---------------------------------- |
| Создать L2-коммутатор          | `Bridge → Bridge/Ports`           | default VLAN, Port Settings        |
| Посмотреть FDB                 | `Bridge → Hosts`                  | `show switch FDB`                  |
| Статический MAC → порт         | `Bridge → Hosts → Add New`        | `add switch filter action=forward` |
| Отключить обучение             | `Bridge → Ports → Learn=no`       | `activate switch port lock`        |
| Ограничить MAC                 | Bridge Filter                     | `learn=N intrusionaction=...`      |
| Статический ARP                | `IP → ARP`                        | `add ip arp`                       |
| Скорость и дуплекс             | `Interfaces → Ethernet`           | `set switch port speed=...`        |
| Зеркалирование                 | `Switch → Switch`                 | `enable/set switch mirror`         |
| Только RX или TX               | не поддерживается RTL8367         | `mirror=rx` или `mirror=tx`        |
| Блокировка неизвестного MAC    | Bridge Filter `drop`              | `intrusionaction=discard`          |
| Отключение порта при нарушении | требуется отдельная автоматизация | `intrusionaction=disable`          |
| Карантинная VLAN               | RADIUS/MAC-auth                   | RADIUS/MAC-auth                    |
